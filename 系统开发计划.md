# 高中全科AI升学成长陪伴系统 - 完整开发计划

> 基于项目三份核心文档整合设计，可直接用于研发团队执行

---

## 一、项目概述与核心定位

### 1.1 项目定位
面向高中生的 **AI原生全自动学情陪伴 + 智能段位成长系统**

**核心宗旨**：
- 重AI智能提分
- 零人工运维
- 轻志愿工具
- 强段位激励

### 1.2 核心差异化价值
| 传统系统痛点 | 本系统解决方案 |
|------------|--------------|
| 需人工维护题库、院校数据 | AI全自动化采集、生成、更新 |
| 学习方案千篇一律 | AI千人千面个性化任务生成 |
| 数据滞后、资源陈旧 | LLM实时动态迭代更新 |
| 学生目标模糊 | 段位可视化 + 差距量化激励 |

### 1.3 标杆用户案例
**高二文科学生（450分→目标600分，数学50分极弱）**
- 所有AI算法、分层逻辑均以此案例做自动化测试校验
- 系统需支持全分数段、全选科组合适配

### 2.2 技术栈选型

#### 前端技术栈
- **框架**: Vue3 + Vite4
- **UI组件库**:
  - Element Plus（管理后台PC端）
  - Vant4（学生/家长移动端）
- **数据可视化**: ECharts 5
  - 总分曲线、位次曲线、知识点热力图、段位进度条
- **网络请求**: Axios + 请求拦截器
- **构建**: Docker镜像打包，支持Linux云服务器部署

#### 后端技术栈
- **核心框架**: SpringBoot 2.7.18
- **权限框架**: SpringSecurity 6 + JWT（单点登录、单端登录限制）
- **ORM**: MyBatis-Plus 3.5.3
- **AI对接**: 自定义LLM调度Client（支持多模型切换）
- **定时任务**: XXL-Job / Spring Scheduled
- **接口规范**: RESTful API，统一返回体、全局异常拦截

#### 数据存储与运维
- **主数据库**: MySQL 8.0
- **缓存**: Redis 6.2
- **文件存储**: 对象存储/本地存储（院校LOGO）
- **备份机制**: 定时Shell脚本每日全量备份，保留30天

## 四、核心功能模块开发任务

### 4.1 AI大模型中枢层（优先级：最高）

#### 任务4.1.1 高考数据智能抓取接口
**功能描述**：定时任务触发LLM，抓取各省教育考试院一分一段数据，清洗结构化写入score_rank表

**技术要点**：
- 使用XXL-Job或Spring Scheduled定时触发
- 对接LLM API进行数据抓取和结构化
- 自动过滤重复、过期数据
- 异常处理和重试机制

**预估工时**: 3-5天

#### 任务4.1.2 院校素材聚合接口
**功能描述**：按需/定时抓取院校官方LOGO、名称、批次，分类存入college_basic表

**技术要点**：
- 设计爬虫策略，遵守robots协议
- 图片存储优化（压缩、CDN）
- 学生首页请求时根据批次随机取3条返回

**预估工时**: 4-6天

#### 任务4.1.3 智能题库生成接口
**功能描述**：根据学生选科、当前分数赛道、薄弱知识点，实时生成对应难度习题

**技术要点**：
- 基于考纲生成题目的Prompt设计
- 难度分层逻辑：
  - ≤500分：60%基础 + 30%中档 + 10%简单压轴
  - 500-580分：均衡分配
  - ≥580分：增加难题占比
- 每周自动迭代题库，淘汰低适配题目

**预估工时**: 7-10天

#### 任务4.1.4 学情分析&任务生成接口
**功能描述**：输入学生历次分数、错题集合，输出薄弱学科标签、每日学习时长权重、当日个性化任务清单

**技术要点**：
- 提分拆分权重算法：得分率<40%极弱学科分配40%时长
- 任务生成千人千面逻辑
- 周/月度复盘报告生成

**预估工时**: 5-7天

#### 任务4.1.5 段位激励文案生成接口
**功能描述**：输入当前分、目标分、心仪院校分差，动态生成极简激励短句、段位总结文案

**技术要点**：
- 避免固定话术库，确保多样性
- 正向激励导向，无压迫性预警
- 文案风格全局可配置

**预估工时**: 2-3天

## 五、开发迭代计划

### 5.1 一期：MVP基础版（快速上线，家庭自用可用）

**开发周期**: 6-8周

**开发范围**：
1. ✅ 登录权限、三角色账号体系、学生档案基础CRUD
2. ✅ 模考录入分数、位次换算、首页三段院校对标卡片、段位进度条
3. ✅ AI基础任务生成、周测记录、成长曲线可视化
4. ✅ 管理员极简账号管理后台

**交付标准**：
- 可录入学生基础信息
- 每周录入考试分数
- 自动展示当前/目标/心仪院校段位差距
- AI生成基础每日学习任务

**技术债务管理**：
- AI接口可先用Mock数据，二期替换真实LLM
- 院校素材可先手动导入初始数据，二期AI自动抓取

### 5.2 二期：完整AI闭环版（全智能无人运维）

**开发周期**: 8-10周

## 六、技术风险与应对策略

### 6.1 AI大模型对接风险

**风险点**：
- LLM API调用失败或不稳定
- 生成内容质量不可控
- 响应延迟影响用户体验

**应对策略**：
1. 多模型备份机制（主模型 + 备用模型）
2. 内容审核和过滤机制
3. Redis缓存高频AI输出
4. 异步生成 + 前端加载动效
5. 关键数据降级方案（如位次换算先用规则，AI失败时回退）

### 6.2 数据抓取合规风险

**风险点**：
- 院校LOGO抓取可能违反版权
- 高考数据来源合法性问题

**应对策略**：
1. 仅使用官方公开数据源
2. LOGO存储使用官方公开资源或自行设计
3. 添加数据来源声明和版权声明
4. 咨询法务确认合规性

### 6.3 系统性能风险

**风险点**：
- AI生成题目、报告耗时较长
- 高并发场景下数据库压力
- 大量历史数据查询性能下降

**应对策略**：
1. 关键数据Redis缓存（位次映射、院校素材、AI高频输出）
2. 数据库索引优化（所有查询字段添加索引）
3. 分页查询限制单次返回数量
4. 异步处理长耗时任务
5. 定期数据归档和清理

### 6.4 数据安全风险

**风险点**：
- 学生隐私数据泄露
- 账号被盗用风险

**应对策略**：
1. 用户密码加密存储（BCrypt）
2. 学情数据加密存储
3. JWT Token有效期控制
4. 单端登录限制
5. 接口防刷机制
6. HTTPS传输加密

## 七、质量保障措施

### 7.1 开发规范

**代码规范**：
- Java代码遵循阿里巴巴Java开发手册
- 前端代码遵循Vue3官方风格指南
- 所有接口必须有注释说明
- 关键业务逻辑必须有单元测试

**Git提交规范**：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- refactor: 重构代码
- test: 测试相关

### 7.2 测试策略

**单元测试**：
- 核心业务逻辑覆盖率 ≥ 80%
- AI算法逻辑必须有单元测试

**集成测试**：
- 所有API接口必须有集成测试
- 权限控制逻辑必须测试

**系统测试**：
- 以标杆案例（450分文科冲600）为主测用例
- 全流程端到端测试

**性能测试**：
- 压力测试：模拟100并发用户
- 接口响应时间：< 2秒（AI生成接口除外）
- 数据库查询优化：慢查询监控

### 7.3 部署和运维

**部署方案**：
- Docker容器化部署
- Nginx反向代理
- HTTPS证书配置
- 日志集中管理

**监控方案**：
- 系统运行状态监控
- 接口健康检查

- 数据库备份状态监控
