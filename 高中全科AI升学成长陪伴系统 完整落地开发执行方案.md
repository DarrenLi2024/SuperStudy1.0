# 高中全科AI升学成长陪伴系统 完整落地开发执行方案

基于两份文档：《AI Native 全智能重构开发规划》\+《全套 UI 界面设计规范》，分**需求校验、技术方案、前后端开发任务拆解、数据库设计、AI 中枢对接、迭代排期、交付标准**全流程落地，可直接交付研发团队开工。

## 一、开发前置校验（对齐 AI Native 核心约束）

### 硬性开发红线（全程强制执行）

1. 全系统**移除所有人工上传、手动录入、素材编辑、题库维护、院校数据修改**功能；高考一分一段、院校库、习题全部由 LLM 自动抓取、生成、更新。

2. 院校展示仅输出**稳妥批次**，随机 3 校（LOGO \+ 校名 \+ 批次），不展示冲 / 保院校、专业、分数线、招生计划。

3. 学生端视觉第一优先级：段位院校对标卡片，其次 AI 每日学习任务，弱化复杂数据。

4. 管理员后台仅保留账号管理、系统监控、基础 AI 参数配置，无任何数据运维页面。

5. 三角色权限强隔离：学生（学习答题）、家长（纯只读查看）、管理员（账号 / 系统运维，不可修改学情资源）。

### 样板业务锚点（文科 450→600，数学极弱）

系统所有算法、分层逻辑、任务权重、段位换算均内置该案例做自动化测试校验。

## 二、整体技术架构落地实现

### 2\.1 分层架构代码实现层级

```Plain Text
前端展示层（Vue3+Vite）
    ↓↑ HTTP/HTTPS + JWT鉴权
接口网关层（SpringBoot Gateway）
    ↓↑
业务逻辑层（SpringBoot核心服务）
    ├─ 用户账号服务（登录、三角色权限、档案管理）
    ├─ AI学情服务（诊断、任务生成、错题分析、复盘报告）
    ├─ 模考位次服务（试卷生成、分数折算、位次匹配、段位计算）
    ├─ 段位激励服务（院校随机匹配、分差计算、进度条数据）
    └─ AI中枢调度服务（对接LLM、资源自治调度）
    ↓↑
数据持久层（MyBatis-Plus + MySQL8.0）
    ↓↑
缓存层（Redis）
    ↓↑
AI自治资源库（LLM自动维护：位次库/院校素材库/动态题库）
```

### 2\.2 完整技术栈落地清单

#### 前端（学生端 \+ 家长端 \+ 管理后台）

1. 框架：Vue3 \+ Vite4

2. UI 组件：Element Plus（管理后台 PC）、Vant4（学生移动端自适应）

3. 可视化：ECharts 5（总分曲线、位次曲线、知识点热力图、段位进度条）

4. 网络：Axios \+ 请求拦截器（统一 JWT、错误捕获、AI 加载状态）

5. 静态资源：院校 LOGO 缓存、本地静态样式，AI 动态资源走接口实时拉取

6. 构建打包：Docker 镜像打包，支持 Linux 云服务器部署

#### 后端 Java 服务

1. 核心框架：SpringBoot 2\.7\.18

2. 权限体系：SpringSecurity6 \+ JWT 单点登录、单端登录限制

3. ORM：MyBatis\-Plus 3\.5\.3（MySQL 持久化）

4. AI 对接模块：自定义 LLM 调度 Client，支持多模型切换（通用大模型 \+ 高考垂直微调模型）

5. 定时任务：XXL\-Job / Spring Scheduled（AI 定时抓取高考数据、刷新院校素材、迭代题库）

6. 接口规范：RESTful，统一返回体、全局异常拦截

#### 数据存储 \& 运维

1. 主数据库：MySQL 8\.0（持久化用户档案、考试记录、错题、打卡、成长数据）

2. 缓存：Redis 6\.2（缓存 LLM 高频输出、位次映射、登录令牌、院校素材）

3. 文件存储：对象存储 / 本地存储（AI 抓取院校 LOGO 自动存储，自动清理失效素材）

4. 备份机制：定时 Shell 脚本每日全量备份 MySQL，保留 30 天备份文件，后台一键恢复

## 三、数据库表设计（分两大块：用户私有持久数据 \+ AI 自治资源表）

### 模块 A：用户私有数据表（永久持久化，不可自动清理）

1. `sys_user` 用户账号表
字段：id、账号、密码加密、角色（student/parent/admin）、绑定学生 id、启用状态、创建时间

2. `student_profile` 学生档案表
字段：id、user\_id、年级、选科组合、高考模式、目标总分、心仪院校名称、心仪院校批次、基线总分、基线等效位次、剩余备考天数（AI 自动计算）

3. `exam_record` 考试记录表
字段：id、student\_id、考试类型（周测 / 月度诊断）、各科分数、总分、等效高考分、等效三年位次、当前稳妥批次、考试时间、AI 诊断报告文本

4. `task_record` AI 每日任务打卡表
字段：id、student\_id、任务日期、任务内容、完成率、AI 点评、完成状态

5. `error_question` 错题归集表
字段：id、student\_id、学科、知识点、题目内容、错误答案、AI 错因分析、补强训练标记

6. `growth_record` 段位升级记录表
字段：id、student\_id、升级前批次、升级后批次、升级时总分、升级时间、AI 激励文案

### 模块 B：AI 自治资源数据表（LLM 定时自动增删改，无人工操作入口）

1. `college_basic` 院校素材自治表
字段：id、院校名称、logo 存储路径、录取批次、科类（物理 / 历史）、抓取更新时间，无人工编辑后台页面

2. `score_rank` 一分一段自治表
字段：年份、科类、分数、对应位次、省份，AI 每年高考后自动抓取更新

3. `ai_question_bank` AI 动态题库表
字段：id、学科、知识点、难度（基础 / 中档 / 难题）、题目、答案、适配分数段标签，定时 AI 淘汰老旧题目、新增题型

## 四、AI 大模型中枢核心开发任务（系统核心驱动层）

### 4\.1 LLM 五大自动化能力接口开发

1. **高考数据解析接口**
定时任务触发 LLM，抓取本省考试院一分一段，清洗结构化写入`score_rank`，自动过滤重复、过期数据。

2. **院校素材聚合接口**
按需 / 定时抓取院校官方 LOGO、名称、批次，分类存入`college_basic`；学生首页请求段位卡片时，根据批次随机取 3 条返回。

3. **智能题库生成接口**
根据学生选科、当前分数赛道、薄弱知识点，实时生成对应难度习题；每周自动批量迭代题库，淘汰低适配题目。

4. **学情分析 \& 任务生成接口**
输入学生历次分数、错题集合，输出：薄弱学科标签、每日学习时长权重、当日个性化任务清单、周 / 月度复盘报告。

5. **段位激励文案生成接口**
输入当前分、目标分、心仪院校分差，动态生成极简激励短句、段位总结文案，无固定话术库。

### 4\.2 AI 约束规则内置代码

- 题库难度配比：≤500 分 60% 基础 \+ 30% 中档 \+ 10% 简单压轴；500\-580 均衡；≥580 增加难题占比

- 稳妥院校匹配阈值：等效位次 ±5000 名内同批次院校随机抽取

- 提分拆分权重：得分率＜40% 极弱学科分配 40% 每日学习时长

## 五、前端页面开发任务（严格遵循 UI 规范，无人工运维入口）

### 5\.1 登录统一页面

- 三角色共用登录表单，登录后根据角色跳转对应端

- 无注册、游客入口，无素材上传、数据录入按钮

### 5\.2 学生端 5 大页面开发

1. **首页（段位总览页，核心页面）**
顶部状态栏 → 三段院校对标卡片 → 段位进度条 → AI 今日任务 → 当日 AI 点评
院校卡片纯 AI 接口渲染，无手动编辑院校功能；进度条 ECharts 动态渲染，段位升级触发动效。

2. **AI 智能学习中心**
AI 专项训练区、错题复盘区、知识点热力图、打卡记录；所有习题由 AI 实时接口返回，无手动导入题库按钮。

3. **AI 全真模考中心**
周测 / 月度模考入口、历史考试列表、考后 AI 诊断弹窗；试卷由 LLM 实时生成，考完自动调用位次换算接口刷新段位。

4. **成长数据中心**
总分趋势、位次曲线、单科曲线、段位升级历史、AI 月度报告；全部图表数据由 AI 统计服务输出。

### 5\.3 家长端唯一页面（纯只读）

仅展示：当前段位、心仪院校分差、本周 AI 点评、学习完成率、成绩趋势；**无任何编辑、提交、上传按钮**。

### 5\.4 管理员后台 2 个页面（AI Native 精简版）

1. 用户账号管理：新增 / 禁用 / 重置学生、家长账号，仅操作账号权限，不可修改学情、院校、题库资源

2. 系统监控页：运行日志、自动备份状态、AI 基础参数配置（赛道阈值、位次差值）、接口健康监控
彻底移除：题库管理、院校素材上传、高考数据录入、考点维护页面。

### 5\.5 全局 UI 交互统一实现

1. AI 加载动效：所有 AI 接口请求时显示光点轻量化加载动画

2. 段位升级动效：进度条填充、卡片高亮缩放、弹窗 AI 激励文案

3. 色彩统一：主科技蓝、进步绿、停滞灰、待办浅橙，圆角卡片统一样式

4. 院校卡片固定组件封装：全局复用，LOGO / 名称 / 批次固定排版，不可自定义修改

## 六、后端业务核心开发任务

### 6\.1 用户权限服务

- JWT 登录鉴权、单端登录校验、角色接口拦截（家长不可访问答题接口、管理员不可访问学生学习接口）

- 学生档案基础信息 CRUD（仅基础年级 / 选科手动录入一次，其余标签 AI 自动更新）

### 6\.2 模考位次换算服务

1. 考试提交后自动折算等效高考总分（适配校内试卷难度加权）

2. 查询`score_rank`表，匹配近三年等效位次

3. 根据位次匹配对应院校批次，调用 AI 院校接口随机返回 3 所稳妥院校

4. 计算当前分与目标分、心仪院校分差，自动生成段位进度条数据

### 6\.3 AI 自适应学习服务

1. 根据学生赛道、薄弱学科自动分配每日任务权重

2. 错题自动归类，AI 分析错因，推送同类补强习题

3. 每周自动生成周复盘、每月生成月度成长报告，持久化存储

### 6\.4 段位激励服务

封装院校卡片统一返回结构：\[\{logo, name, batch\}\]，每次请求随机打乱同批次院校；实时计算心仪院校总分缺口、单科缺口。

### 6\.5 定时自治任务（无人工干预）

1. 每日凌晨：AI 同步本省最新一分一段数据

2. 每周：AI 批量刷新院校素材、迭代题库淘汰老旧题目

3. 每日：MySQL 全量自动备份

## 七、分三期迭代开发排期（可直接用于研发排期）

### 一期 MVP 基础版（最快上线，家庭自用可用）

开发范围：

1. 登录权限、三角色账号体系、学生档案基础 CRUD

2. 模考录入分数、位次换算、首页三段院校对标卡片、段位进度条

3. AI 基础任务生成、周测记录、成长曲线可视化

4. 管理员极简账号管理后台
交付标准：可录入学生基础信息，每周录入考试分数，自动展示当前 / 目标 / 心仪院校段位差距，AI 生成基础每日学习任务。

### 二期 完整 AI 闭环版（全智能无人运维）

开发范围：

1. LLM 全链路自动化：自动抓取位次、院校素材、动态生成题库

2. AI 错题智能补强、知识点热力图谱、周 / 月度全自动复盘报告

3. 完整模考系统（AI 自动生成试卷，线上答题）

4. 家长端完整监督页面、段位升级动效、全量激励文案生成
交付标准：完全零人工运维，无需手动更新任何高考 / 院校 / 题库数据，形成完整日 \- 周 \- 月学习闭环。

### 三期 成熟优化版

开发范围：

1. 多学生批量档案管理、AI 学习停滞智能预警

2. 图表视觉优化、批量导出成长数据

3. LLM 模型切换配置、AI 生成内容缓存优化

4. 全套自动化运维监控告警
交付标准：支持多学生、班级批量使用，系统稳定低维护，AI 自适应逻辑高度完善。

## 九、系统核心合规 \& 质量校验标准（开发自测必过）

1. 页面无任何「上传、导入、新增素材、手动录入题库 / 院校数据」按钮、弹窗、菜单

2. 所有院校、习题、高考数据修改 / 新增操作仅由定时 AI 任务完成，无人工入口

3. 院校展示仅随机稳妥批次 3 所，无冲刺、保底院校展示逻辑

4. 三角色权限隔离：家长无法答题、管理员无法修改学生学情、学生无法访问后台

5. 每次模考自动更新等效位次、院校卡片、段位进度条、心仪院校分差

6. 系统所有报告、任务、激励文案全部由 LLM 实时生成，无固定人工配置文本库
