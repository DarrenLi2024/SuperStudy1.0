# 贡献指南

> 欢迎为高中全科AI升学成长陪伴系统贡献代码！

---

## 一、贡献流程

### 1.1 开始贡献

1. **阅读文档**：阅读《AI快捷执行手册》了解任务
2. **检查状态**：确认任务状态和依赖
3. **开始任务**：更新任务状态为 `in_progress`
4. **执行开发**：按照任务卡片要求开发
5. **更新状态**：完成后更新任务状态和执行日志
6. **等待审计**：等待审计通过

### 1.2 任务状态

| 状态 | 说明 |
|-----|------|
| `pending` | 待开始 |
| `in_progress` | 进行中 |
| `done` | 完成 |
| `audited` | 已审计 |
| `blocked` | 阻塞 |

## 二、开发规范

### 2.1 代码规范

#### 后端规范

- ✅ 使用Java 8语法
- ✅ 遵循SpringBoot最佳实践
- ✅ 使用MyBatis-Plus进行数据库操作
- ✅ 统一响应格式（ResponseResult）
- ✅ 密码使用BCrypt加密
- ✅ 使用JWT进行认证

#### 前端规范

- ✅ 使用Vue 3 Composition API
- ✅ 使用Element Plus/Vant组件库
- ✅ 使用Axios封装API调用
- ✅ 使用Pinia进行状态管理
- ✅ 使用Vue Router进行路由管理
- ✅ 组件化开发，复用性强

### 2.2 命名规范

| 类型 | 规范 | 示例 |
|-----|------|-----|
| 类名 | PascalCase | `UserController` |
| 方法名 | camelCase | `getUserById` |
| 变量名 | camelCase | `userId` |
| 常量名 | UPPER_SNAKE_CASE | `MAX_PAGE_SIZE` |
| 包名/目录名 | lowercase | `com/example/sys` |
| 组件名 | PascalCase | `CollegeCard.vue` |

### 2.3 提交规范

#### 提交信息格式

```
[任务ID] 描述信息

详细说明（可选）
```

#### 提交类型

| 类型 | 说明 |
|-----|------|
| `feat` | 新功能 |
| `fix` | Bug修复 |
| `docs` | 文档更新 |
| `refactor` | 代码重构 |
| `test` | 测试添加 |
| `chore` | 构建/工具更新 |

#### 示例

```
[TASK-BE01] 实现用户登录接口

- 添加登录API
- 实现JWT令牌生成
- 添加密码验证逻辑
```

## 三、代码审查

### 3.1 审查流程

1. **提交代码**：完成任务后提交代码
2. **审查代码**：由测试AI进行代码审查
3. **修复问题**：根据审查意见修复问题
4. **重新提交**：修复后重新提交
5. **通过审查**：审查通过后更新任务状态

### 3.2 审查要点

| 检查项 | 检查内容 |
|-------|---------|
| 代码质量 | 方法职责、代码复用、复杂度 |
| 安全合规 | 密码加密、SQL注入防护、输入校验 |
| 命名规范 | 类名、方法名、变量名符合规范 |
| 错误处理 | 异常处理、日志记录 |
| 性能问题 | N+1查询、未使用缓存 |
| 文档完备 | 代码注释、接口文档 |

## 四、Bug报告

### 4.1 Bug报告格式

```
【Bug ID】BUG-001
【模块】用户权限服务
【描述】登录接口密码错误时返回500错误
【复现步骤】
1. 访问登录接口
2. 输入正确手机号和错误密码
3. 接口返回500错误
【预期结果】返回400错误，提示密码错误
【修复方案】在Service层捕获密码不匹配异常，返回正确错误码
【状态】待修复/修复中/已修复
```

### 4.2 Bug修复流程

1. **确认Bug**：确认Bug描述和复现步骤
2. **定位问题**：定位问题代码
3. **编写测试**：编写测试用例验证Bug
4. **修复代码**：修复代码
5. **运行测试**：运行测试确认修复成功
6. **更新状态**：更新Bug状态为已修复

## 六、开发红线

以下规则为开发红线，必须严格遵守：

- ✅ **零人工运维**：系统上线后无需人工维护任何数据，无任何上传/录入/编辑入口
- ✅ **仅稳妥院校对标**：随机返回3所同批次稳妥院校，无冲保院校展示
- ✅ **三角色权限强隔离**：家长只读、管理员仅账号管理，无数据运维权限
- ✅ **无注册入口**：系统无注册功能，账号由管理员创建
- ✅ **无游客入口**：系统无游客访问权限
- ✅ **无个性化配置**：除基础信息外，无用户自定义配置入口
