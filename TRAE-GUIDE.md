## 四、集成流程

### 4.1 接口联调流程

1. **阅读接口文档**：查看《API接口契约规范》了解接口定义
2. **检查接口状态**：确认后端接口已完成开发
3. **执行接口测试**：使用Postman测试各接口
4. **记录问题**：发现问题记录到问题清单
5. **协调修复**：通知对应AI修复问题
6. **验证修复**：重新测试，确认问题解决

### 4.2 端到端测试流程

1. **编写测试用例**：覆盖核心业务流程
2. **执行测试**：使用Playwright执行端到端测试
3. **分析结果**：分析测试失败原因
4. **定位问题**：确定问题属于前端还是后端
5. **协调修复**：通知对应AI修复
6. **回归测试**：重新执行测试

## 五、问题排查规范

### 5.1 问题类型

| 类型 | 描述 | 排查方向 |
|-----|------|---------|
| 接口错误 | API返回错误码 | 检查后端代码、数据库、日志 |
| 数据不一致 | 前后端数据不一致 | 检查接口响应、前端处理逻辑 |
| 性能问题 | 页面加载慢 | 检查API响应时间、前端渲染 |
| 权限问题 | 无权限访问 | 检查JWT Token、角色权限 |
| 跨域问题 | CORS错误 | 检查跨域配置 |

### 5.2 问题记录格式

```
【问题ID】ISSUE-001
【类型】接口错误
【模块】用户权限服务
【描述】登录接口返回500错误
【复现步骤】
1. 调用POST /api/v1/sys/login
2. 输入正确手机号和密码
3. 接口返回500错误
【日志信息】数据库连接超时
【排查结果】数据库连接池配置错误
【修复方案】调整数据库连接池参数
【负责人】Claude Code
【状态】待修复
```

### 5.3 问题处理流程

1. **发现问题**：执行测试时发现问题
2. **记录问题**：按格式记录到问题清单
3. **分析问题**：分析问题原因和影响范围
4. **分配任务**：通知对应AI修复
5. **跟踪进度**：跟踪问题修复进度
6. **验证修复**：验证问题是否解决
7. **关闭问题**：更新问题状态为已关闭
